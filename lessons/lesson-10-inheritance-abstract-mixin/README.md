# Lesson 10 — Inheritance, Abstract Classes & Mixins

## Topics Covered
- `extends` for inheritance
- `super` to call parent constructor
- `@override` to override parent methods
- `abstract` classes — cannot be instantiated directly
- `implements` — implement an interface
- `mixin` — reusable blocks of behavior
- `with` — apply a mixin to a class

## Flutter Connection
Flutter's widget system is entirely inheritance-based. `StatelessWidget` and `StatefulWidget` are abstract classes you `extend`. Many Flutter packages use `mixin` for shared behavior like `TickerProviderStateMixin` for animations.

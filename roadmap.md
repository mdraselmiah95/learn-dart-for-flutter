# Learning Roadmap

> Dart beginner ‚Üí standard level ‚Üí Flutter ready

---

## Stage 1 ‚Äî Beginner (Lessons 01‚Äì02)

**Goal:** Get Dart installed, write your first program, understand variables.

- Install Dart SDK
- Understand what Dart is and why it exists
- Write `Hello, World!`
- Learn comments, print, and basic syntax
- Understand all variable types: `var`, `String`, `int`, `double`, `bool`, `dynamic`, `final`, `const`
- Understand `null`, `late`, and nullable types

**After this stage you can:** Write and run a basic Dart program, declare any type of variable.

---

## Stage 2 ‚Äî Core Dart (Lessons 03‚Äì05)

**Goal:** Control what your program does and how it repeats.

- Use all operators: arithmetic, comparison, logical, null-aware
- Write conditions with `if / else / switch / ternary`
- Loop with `for`, `while`, `do-while`, `for-in`
- Use `break` and `continue`

**After this stage you can:** Write logic that makes decisions and repeats actions ‚Äî the backbone of any app.

---

## Stage 3 ‚Äî Functions & Collections (Lessons 06‚Äì07)

**Goal:** Organize code into reusable blocks, work with groups of data.

- Write functions with positional and named parameters
- Use arrow functions and anonymous functions
- Pass functions as arguments (higher-order functions)
- Create and use `List`, `Map`, and `Set`
- Use collection methods: `map`, `where`, `reduce`, `forEach`, `sort`

**After this stage you can:** Write clean, reusable Dart code. Understand how Flutter widget trees use functions.

---

## Stage 4 ‚Äî Null Safety (Lesson 08)

**Goal:** Write safe code that never crashes from null errors.

- Understand why null safety exists
- Use `?` to allow null values
- Use `??` for fallback values
- Use `?.` for safe method calls
- Use `!` only when certain (and understand the risk)
- Use `late` for deferred initialization

**After this stage you can:** Write null-safe Dart ‚Äî required for all modern Flutter apps.

---

## Stage 5 ‚Äî OOP (Lessons 09‚Äì10)

**Goal:** Organize your code using classes and objects ‚Äî the way Flutter is built.

- Create classes with properties and methods
- Write constructors including named and factory constructors
- Use inheritance with `extends`
- Use `abstract` classes and `implements`
- Use `mixins` for shared behavior
- Understand `@override`

**After this stage you can:** Model real-world data in code. Understand how Flutter widgets are classes.

---

## Stage 6 ‚Äî Async Programming (Lesson 11)

**Goal:** Handle tasks that take time ‚Äî network calls, file reads, timers.

- Understand why async exists (single-threaded Dart)
- Use `Future` for one-time async values
- Use `async` and `await` to write clean async code
- Handle errors in async code with `try/catch`
- Understand `Stream` for continuous async data

**After this stage you can:** Call APIs, read files, and handle loading states in Flutter.

---

## Stage 7 ‚Äî Flutter Readiness (Lesson 12)

**Goal:** Bridge the gap between Dart knowledge and Flutter code.

- Create data models (plain Dart classes)
- Parse and build JSON
- Write service classes for API calls
- Understand the patterns Flutter uses: state, models, services
- Build a small Dart app that mirrors Flutter app logic

**After this stage you are:** Ready to start learning Flutter.

---

## Stage 8 ‚Äî Mini Projects

**Goal:** Apply everything you learned by building real logic.

| Project | Skills Used |
|---------|-------------|
| Calculator | Functions, operators, control flow |
| Todo Logic App | List, Map, OOP, CRUD |
| Expense Tracker | Classes, collections, data modeling |
